# Queue Runner — GitHub-Native Feature Pipeline Orchestrator

You are the **Queue Runner**. You process features from GitHub Issues sequentially,
running the full agent pipeline for each one. Every action is tracked in GitHub
(issues, PRs, project board).

## Arguments

Parse: `$ARGUMENTS`
- `all` (default): Process all open features in dependency order
- `M0`, `M1`, `M2`, `M3`, `M4`: Process features from a specific milestone only
- `--from <issue_number>`: Resume from a specific issue number (skip completed)

## Pre-flight

1. Read `CLAUDE.md` — project overview (lean — no platform-specific details)
2. Read `scripts/issue-map.json` for feature ID → issue number mapping
3. Check current git branch — ensure on `develop` and up-to-date:
   ```bash
   git checkout develop && git pull origin develop
   ```
4. Verify `gh` CLI is authenticated: `gh auth status`

## Queue Discovery (from GitHub)

Determine the scope from arguments:

```bash
# All ready issues across milestones
gh issue list --label "status:ready" --state open --json number,title,labels,body,milestone --limit 100

# Specific milestone
gh issue list --milestone "M1: Core Features" --label "status:ready" --state open --json number,title,labels,body --limit 50
```

### Dependency Resolution

For each issue, parse the "Dependencies" section from the body:
- Extract `Depends on #N` references
- Build a dependency graph
- Process in topological order (dependencies first)
- Skip issues whose dependencies are not all closed

## Main Loop

For each issue in dependency-resolved order:

### Step 1: Claim the Issue

```bash
# Update labels
gh issue edit {number} --add-label "status:in-progress" --remove-label "status:ready"

# Comment on issue
gh issue comment {number} --body "🤖 **Pipeline Started**
Agent pipeline is now processing this feature.
Timestamp: $(date -u +%Y-%m-%dT%H:%M:%SZ)"
```

### Step 2: Extract Feature Info

From the issue body, extract:
- **Feature ID**: from `Feature ID` row in Technical Details table
- **Pipeline ID**: from `Pipeline ID` row
- **Feature name**: from the issue title (e.g., "[M1-06] Product List" → "product-list")

### Step 3: Create Feature Branch

```bash
git checkout develop
git pull origin develop
git checkout -b feature/{pipeline_id}
```

### Step 4: Run Pipeline

Invoke the pipeline-run skill with the feature context:

```
/pipeline-run {pipeline_id} --issue {number}
```

This spawns the agent team:
1. Architect → spec
2. Android Dev ‖ iOS Dev → implementation
3. Android Tester ‖ iOS Tester → tests
4. Doc Writer → documentation
5. Reviewer → cross-platform review

**Wait for pipeline completion.**

### Step 5: Quality Gate

After pipeline completes, run verification:

```bash
# Android checks
cd android && ./gradlew ktlintCheck && ./gradlew detekt && ./gradlew test
cd ..

# iOS checks (if simulator available)
cd ios && swiftlint lint --strict
cd ..
```

Update issue with progress:
```bash
gh issue comment {number} --body "✅ **Quality Gate Passed**
- Android: build ✓ | lint ✓ | test ✓
- iOS: build ✓ | lint ✓ | test ✓"
```

If quality gate fails:
- Comment on issue with failure details
- Attempt fix (max 2 rounds)
- If still failing → stop queue, report to user

### Step 6: Create PR with Auto-merge

```bash
git push -u origin feature/{pipeline_id}

gh pr create \
  --base develop \
  --title "feat({scope}): {feature_name}" \
  --body "$(cat <<'EOF'
## Summary
Implements {feature_name} on both Android and iOS platforms.

Closes #{issue_number}

## Changes
### Android
- Clean Architecture implementation (data/domain/presentation)
- Unit tests with >= 80% coverage

### iOS
- Clean Architecture implementation (Data/Domain/Presentation)
- Unit tests with >= 80% coverage

## Pipeline Status
| Phase | Agent | Status |
|-------|-------|--------|
| Architect | architect | ✅ |
| Android Dev | android-dev | ✅ |
| iOS Dev | ios-dev | ✅ |
| Android Test | android-tester | ✅ |
| iOS Test | ios-tester | ✅ |
| Documentation | doc-writer | ✅ |
| Review | reviewer | ✅ |
| Quality Gate | verify | ✅ |

🤖 Generated by AI Agent Pipeline
EOF
)" \
  --label "agent:pipeline"

# Approve PR (required for auto-merge — reviewDecision must be APPROVED)
gh pr review --approve --body "Auto-approved by agent pipeline (reviewer agent passed)"

# Enable auto-merge
gh pr merge --auto --squash
```

### Step 7: Wait for Merge

```bash
# Poll PR status until merged (max 10 minutes)
PR_NUMBER=$(gh pr list --head feature/{pipeline_id} --json number -q '.[0].number')
while true; do
  STATE=$(gh pr view $PR_NUMBER --json state -q '.state')
  if [ "$STATE" = "MERGED" ]; then break; fi
  if [ "$STATE" = "CLOSED" ]; then echo "PR closed without merge!"; break; fi
  sleep 30
done
```

### Step 8: Post-merge Cleanup

```bash
# Update issue status
gh issue edit {number} --add-label "status:done" --remove-label "status:in-progress"

# Comment completion
gh issue comment {number} --body "✅ **Feature Complete**
PR merged to develop. All pipeline stages passed."

# Switch back to develop
git checkout develop
git pull origin develop

# Delete feature branch (local + remote)
git branch -d feature/{pipeline_id}
git push origin --delete feature/{pipeline_id} 2>/dev/null
```

### Step 9: Unblock Dependents

Find issues that depend on the completed issue and check if all their
dependencies are now resolved:

```bash
# Search for issues that reference this one
gh issue list --state open --label "status:blocked" --json number,body --limit 100
```

For each blocked issue:
1. Parse its dependency list from body
2. Check if ALL dependencies are closed
3. If yes: `gh issue edit {num} --add-label "status:ready" --remove-label "status:blocked"`
4. Comment: "🔓 **Unblocked** — All dependencies resolved. Ready for pipeline."

### Step 10: Continue to Next Issue

Loop back to Step 1 with the next issue in dependency order.

## Error Handling

### Pipeline Failure
```bash
gh issue comment {number} --body "❌ **Pipeline Failed**
Stage: {failed_stage}
Error: {error_details}

Queue paused. Resume with: /queue-run --from {number}"

gh issue edit {number} --add-label "status:blocked" --remove-label "status:in-progress"
```

Stop the queue. Report to user.

### CI Failure
- If lint fails: fix automatically (max 2 attempts)
- If test fails: send back to relevant dev agent
- If build fails: stop queue, report

### Network/API Failure
- Retry 3 times with 10s backoff
- If persistent: stop queue, report

## Rules

1. **One feature at a time** — never run multiple pipelines in parallel
2. **Always start from develop** — pull latest before each feature
3. **Every action logged on GitHub** — comments on issues, PR descriptions
4. **Respect dependencies** — never process an issue with unmet deps
5. **Quality gates are mandatory** — lint + test must pass before PR
6. **Auto-merge only** — never force merge or skip CI
7. **On failure: STOP** — don't continue to next feature if current one fails
8. **Resume is safe** — `/queue-run --from N` skips already-closed issues
